# 2026-07 Changelog Archive

## 2026-07-01 (Codex - io_uring benchmark artifact workflow)

### 작업 단위
- D147 기준으로 Linux `io_uring` benchmark artifact 를 수집하는 수동 GitHub Actions workflow 를 추가했다.

### 변경 내용
- `.github/workflows/iouring-benchmark-artifacts.yml`:
  `workflow_dispatch` 전용 `ubuntu-latest` workflow 를 추가했다.
  workflow 는 `HPS_BENCHMARK_RUNNER_ID=ci-linux-iouring-x64-01`,
  `HPS_BENCHMARK_RUNNER_KIND=ci`를 고정하고 TCP/UDP 각각
  `--baseline-suite ... --runs 1 --protocol <tcp|udp> --backend iouring`을 실행한다.
- TCP/UDP output root 를 분리했다.
  TCP는 `artifacts/benchmarks/runners/ci-linux-iouring-x64-01/<yyyy-mm-dd>/tcp/session-01/`,
  UDP는 같은 date root 아래 `udp/session-01/`에 raw report 와 summary 를 쓴다.
  protocol 별 `history.json`/`history.md`도 각각 protocol root 에 둔다.
- workflow 는 baseline command 가 실패해도 가능한 범위에서 summary/history/root summary/upload 를 계속 수행한 뒤,
  마지막 failure gate 에서 저장해 둔 exit code 를 job failure 로 복원한다.
- `tests/Hps.Benchmarks.Tests/BenchmarkArtifactWorkflowTests.cs`에 workflow static contract 테스트를 추가했다.
  테스트는 Linux 수동 trigger, runner identity, TCP/UDP command, summary/history command, artifact upload,
  final failure gate 순서를 고정한다.
- `docs/superpowers/specs/2026-07-01-iouring-benchmark-artifact-workflow-design.md`와
  `docs/superpowers/plans/2026-07-01-iouring-benchmark-artifact-workflow.md`에 D147 설계와 구현 계획을 기록했다.

### 검증

  실행에서 새 workflow 파일 부재로 new tests 2개가 실패하는 것을 확인했다.

  upload artifact 의 TCP/UDP raw report, summary, history, root summary 를 검토하는 것이다.
