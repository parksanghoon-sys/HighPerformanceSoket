# .claude/review/ — 설계·코드 검토 기록

이 폴더에는 **Claude 가 작성한 검토서**가 쌓인다. 구현은 Codex 가 하고, 검토는 Claude 가 한다
(역할 분리는 [`../../AGENTS.md`](../../AGENTS.md) §6 참조).

## 현재 조치 현황
- [`review-status-2026-06-11.md`](./review-status-2026-06-11.md): 현재 작업 트리 기준으로
  기존 Claude 검토 의견의 조치 여부와 아직 유효한 비차단 항목을 정리한다.
- 기존 검토서 원문은 작성 당시 커밋/작업 상태의 스냅샷으로 보존한다. 현재 구현 상태와 어긋난
  오래된 평가는 위 조치 현황 문서에서 superseded 로 해석한다.

## 파일 규약
- 파일명: `phase<N>-<주제>.md` (예: `phase1-bipbuffer.md`).
- 한 Phase 에 검토가 여러 번이면 `phase1-bipbuffer-r2.md` 처럼 회차를 붙인다.

## 검토서 형식
각 검토서는 다음을 포함한다:
1. **대상**: 검토한 파일/커밋 범위.
2. **요약 판정**: `승인` / `조건부 승인(must-fix 후)` / `반려`.
3. **must-fix**: 다음 작업 전에 반드시 고칠 항목(정확성·불변식 위반). 근거 포함.
4. **should-fix**: 고치면 좋은 항목(성능·가독성).
5. **확인 필요(질문)**: 의도가 모호해 사람/구현자 확인이 필요한 점.

## 처리 흐름
- Codex 는 must-fix 를 해소한 뒤 다음 Phase 로 진행한다.
- 재검토가 필요하면 새 회차 파일을 추가한다.
